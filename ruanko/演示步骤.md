# Ruanko DBMS 终端功能演示步骤

## 准备工作

1. 打开终端
2. 进入项目目录：
   ```bash
   cd "/Users/aoxiang/Desktop/数据库作业/ruanko"
   ```
3. 确保已编译 CLI（如果还没有编译）：
   ```bash
   export GOCACHE="$(pwd)/gocache"
   export GOTMPDIR="$(pwd)/tmp"
   mkdir -p gocache tmp
   go build -o ruanko_cli cmd/ruanko/main.go
   ```
4. 清理旧数据，确保演示环境干净：
   ```bash
   rm -rf dbms_root
   ```
5. 启动程序：
   ```bash
   ./ruanko_cli
   ```

---

## 第 1 步：查看系统数据库

**输入：**
```sql
SHOW DATABASES;
```

**预期效果：** 显示系统自带的 `Ruanko` 系统数据库。

---

## 第 2 步：创建用户数据库

**输入：**
```sql
CREATE DATABASE shop;
```

**预期效果：** 提示 `Database 'shop' created`。

---

## 第 3 步：切换数据库

**输入：**
```sql
USE shop;
```

**预期效果：** 提示 `Database changed to 'shop'`。

---

## 第 4 步：创建表（覆盖全部 5 种数据类型）

**输入：**
```sql
CREATE TABLE products (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price DOUBLE,
    stock INTEGER DEFAULT 0,
    created_at DATETIME,
    is_active BOOL
);
```

**预期效果：** 提示 `Table 'products' created`。

---

## 第 5 步：查看当前数据库的表

**输入：**
```sql
SHOW TABLES;
```

**预期效果：** 显示 `products` 表，记录数为 0，字段数为 6。

---

## 第 6 步：插入数据（覆盖所有数据类型）

**输入：**
```sql
INSERT INTO products VALUES
(1, 'iPhone 15', 5999.00, 100, '2024-01-15 10:30:00', true),
(2, 'MacBook Pro', 14999.00, 50, '2024-02-20 14:00:00', true),
(3, 'AirPods Pro', 1999.00, 200, '2024-03-10 09:00:00', true),
(4, 'iPad Mini', 3999.00, 0, '2024-01-20 11:00:00', false);
```

**预期效果：** 提示 `4 row(s) inserted`。

---

## 第 7 步：全表查询

**输入：**
```sql
SELECT * FROM products;
```

**预期效果：** 以表格形式显示 4 条记录，包含 id、name、price、stock、created_at、is_active 六列。

---

## 第 8 步：条件查询 - 比较运算

**输入：**
```sql
SELECT name, price FROM products WHERE price > 5000;
```

**预期效果：** 返回 price 大于 5000 的商品（iPhone 15、MacBook Pro）。

**继续输入：**
```sql
SELECT * FROM products WHERE stock = 0;
```

**预期效果：** 返回库存为 0 的商品（iPad Mini）。

---

## 第 9 步：条件查询 - 逻辑运算 AND

**输入：**
```sql
SELECT name, price, stock FROM products WHERE stock > 0 AND price < 10000;
```

**预期效果：** 返回库存大于 0 且价格小于 10000 的商品（iPhone 15、AirPods Pro）。

---

## 第 10 步：条件查询 - 逻辑运算 OR

**输入：**
```sql
SELECT * FROM products WHERE stock = 0 OR is_active = false;
```

**预期效果：** 返回库存为 0 或未激活的商品（iPad Mini）。

---

## 第 11 步：更新记录

**输入：**
```sql
UPDATE products SET stock = 150 WHERE id = 1;
```

**预期效果：** 提示 `1 row(s) updated`。

**继续输入：**
```sql
UPDATE products SET price = 13999.00, stock = 80 WHERE id = 2;
```

**预期效果：** 提示 `1 row(s) updated`。

**验证更新结果：**
```sql
SELECT * FROM products WHERE id = 1 OR id = 2;
```

**预期效果：** 显示 id 为 1 和 2 的记录，stock 和 price 已更新。

---

## 第 12 步：删除记录

**输入：**
```sql
DELETE FROM products WHERE id = 4;
```

**预期效果：** 提示 `1 row(s) deleted`。

**验证删除结果：**
```sql
SELECT * FROM products;
```

**预期效果：** 只剩下 3 条记录（id 为 4 的 iPad Mini 已消失）。

---

## 第 13 步：修改表结构 - 添加字段

**输入：**
```sql
ALTER TABLE products ADD COLUMN category VARCHAR(50);
```

**预期效果：** 提示 `Column 'category' added to table 'products'`。

---

## 第 14 步：修改表结构 - 删除字段

**输入：**
```sql
ALTER TABLE products DROP COLUMN category;
```

**预期效果：** 提示 `Column 'category' dropped from table 'products'`。

---

## 第 15 步：索引管理（框架演示）

**输入：**
```sql
CREATE INDEX idx_price ON products (price);
```

**预期效果：** 提示索引创建成功（简化实现）。

**继续输入：**
```sql
DROP INDEX idx_price;
```

**预期效果：** 提示索引删除成功。

---

## 第 16 步：事务控制（简化版）

**输入：**
```sql
BEGIN;
```

**预期效果：** 提示 `Transaction started (simplified)`。

**在事务中插入数据：**
```sql
INSERT INTO products VALUES (5, 'Apple Watch', 2999.00, 300, '2024-04-01 08:00:00', true);
```

**验证事务中数据可见：**
```sql
SELECT * FROM products WHERE id = 5;
```

**预期效果：** 显示刚插入的 Apple Watch。

**回滚事务：**
```sql
ROLLBACK;
```

**预期效果：** 提示 `Transaction rolled back (simplified)`。

**验证回滚后数据已消失：**
```sql
SELECT * FROM products WHERE id = 5;
```

**预期效果：** 返回 0 行（事务中的插入已被回滚）。

---

## 第 17 步：用户管理（简化版）

**输入：**
```sql
CREATE USER admin PASSWORD 'admin123';
```

**预期效果：** 提示 `User 'admin' created`。

**继续输入：**
```sql
GRANT SELECT, INSERT ON products TO admin;
```

**预期效果：** 提示授权成功。

---

## 第 18 步：数据库维护（简化版）

**输入：**
```sql
BACKUP DATABASE shop TO './backup';
```

**预期效果：** 提示备份成功（简化实现）。

**继续输入：**
```sql
RESTORE DATABASE shop FROM './backup';
```

**预期效果：** 提示恢复成功（简化实现）。

---

## 第 19 步：删除表

**输入：**
```sql
DROP TABLE products;
```

**预期效果：** 提示 `Table 'products' dropped`。

**验证表已删除：**
```sql
SHOW TABLES;
```

**预期效果：** 显示 0 个表。

---

## 第 20 步：删除数据库

**输入：**
```sql
DROP DATABASE shop;
```

**预期效果：** 提示 `Database 'shop' dropped`。

**验证数据库已删除：**
```sql
SHOW DATABASES;
```

**预期效果：** 只剩下 `Ruanko` 系统数据库。

---

## 第 21 步：退出程序

**输入：**
```
.exit
```

**预期效果：** 提示 `Bye.` 并退出到终端。

---

## 演示内容总结

| 步骤 | 演示功能 |
|------|---------|
| 1-3 | 数据库的创建、查看、切换 |
| 4-5 | 表的创建、查看（覆盖 5 种数据类型） |
| 6 | 批量插入数据 |
| 7 | 全表查询（SELECT *） |
| 8 | 比较运算（>, =, <=） |
| 9-10 | 逻辑运算（AND, OR） |
| 11 | 更新记录（UPDATE ... SET ... WHERE） |
| 12 | 删除记录（DELETE WHERE） |
| 13-14 | 修改表结构（ADD COLUMN / DROP COLUMN） |
| 15 | 索引管理（CREATE INDEX / DROP INDEX） |
| 16 | 事务控制（BEGIN / ROLLBACK） |
| 17 | 用户与权限（CREATE USER / GRANT） |
| 18 | 数据库维护（BACKUP / RESTORE） |
| 19-20 | 表和数据库的删除 |

以上步骤涵盖了需求文档中 A/B/C 三级的大部分核心功能，适合在终端中逐步展示。
